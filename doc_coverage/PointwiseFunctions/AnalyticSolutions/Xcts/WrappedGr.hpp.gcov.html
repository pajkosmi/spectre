     202             :  *   \text{,}
     203             :  * \f}
     204             :  *
     205             :  * so the spatial metric of the `GrSolution` is used as conformal metric,
     206             :  * \f$\bar{\gamma}_{ij = \gamma_{ij}\f$. This is particularly useful for
     207             :  * superpositions, because it means that the superposed conformal metric of two
     208             :  * `WrappedGr` solutions is probably a good conformal background to solve for a
     209             :  * binary solution (see Xcts::AnalyticData::Binary).
     210             :  *
     211             :  * For example, when the `GrSolution` is `gr::Solutions::KerrSchild`, the
     212             :  * conformal metric is the spatial Kerr metric in Kerr-Schild coordinates and
     213             :  * \f$\psi = 1\f$. It is also possible to
     214             :  * choose a different \f$\psi\f$ so the solution is non-trivial in this
     215             :  * variable, though that is probably only useful for testing and currently not
     216             :  * implemented. It should be noted, however, that the combination of
     217             :  * \f$\psi=1\f$ and apparent-horizon boundary conditions poses a hard problem to
     218             :  * the nonlinear solver when starting at a flat initial guess. This is because
     219             :  * the strongly-nonlinear boundary-conditions couple the variables in such a way
     220             :  * that the solution is initially corrected away from \f$\psi=1\f$ and is then
     221             :  * unable to recover. A conformal-factor profile such as \f$\psi=1 +
     222             :  * \frac{M}{2r}\f$ (resembling isotropic coordinates) resolves this issue. In
     223             :  * production solves this is not an issue because we choose a much better
     224             :  * initial guess than flatness, such as a superposition of Kerr solutions for
     225             :  * black-hole binary initial data.

     231             :  * quantities are not treated dynamically in the source terms when solving the
     232             :  * XCTS equations. If the GR quantities satisfy the Einstein constraints (as is
     233             :  * the case if the `GrSolution` is actually a solution to the Einstein
