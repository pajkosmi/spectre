      32             :  *
      33             :  * \details Given a point with cartesian coordinates \f$\xi^i\f$, let the polar
      34             :  * coordinates \f$(r, \theta, \phi)\f$ with respect to a center \f$x_c^i\f$ be

      36             :  * \xi^0 - x_c^0 &= r sin(\theta) cos(\phi)\\
      37             :  * \xi^1 - x_c^1 &= r sin(\theta) sin(\phi)\\
      38             :  * \xi^2 - x_c^2 &= r cos(\theta)
      39             :  * \f}
      40             :  * The shape map distorts the distance \f$r\f$ between the point and the center
      41             :  * while leaving the angles \f$\theta\f$, \f$\phi\f$ between them preserved by
      42             :  * applying a spherical harmonic expansion with time-dependent coefficients
      43             :  * \f$\lambda_{lm}(t)\f$. There are two ways to specify the time-dependent
      44             :  * coefficients \f$\lambda_{lm}(t)\f$:
      45             :  *
      46             :  * 1. A single FunctionOfTime which specifies all coefficients. This
      47             :  *    FunctionOfTime should have `ylm::Spherepack::spectral_size()` number of
      48             :  *    components. These are in Spherepack order and should be the Spherepack
      49             :  *    coefficients, *not* the spherical harmonic coefficients. See the note
      50             :  *    below. To use this, set the `size_function_of_time_name` argument of the
      51             :  *    constructor to `std::nullopt`.
      52             :  * 2. Two different FunctionOfTime%s. The first is similar to 1.) in that it
      53             :  *    should have the same number of components, be in Spherepack order, and be
      54             :  *    the Spherepack coefficients. The only difference is that the \f$l = 0\f$
      55             :  *    coefficient should be identically 0. The second FunctionOfTime should have
      56             :  *    a single component which will be the \f$l = 0\f$ coefficient. This
      57             :  *    component should be stored as the spherical harmonic coefficient and *not*
      58             :  *    a Spherepack coefficient. See the note below. To use this method, set the
      59             :  *    `size_function_of_time_name` argument of the constructor to the name of
      60             :  *    the FunctionOfTime that's in the cache. This method is useful if we have
      61             :  *    control systems because we have a separate control system controlling a
      62             :  *    separate function of time for the \f$l = 0\f$ coefficient than we do for
      63             :  *    the other coefficients.
      64             :  *
      65             :  * \note The quantities stored in the "shape" FunctionOfTime (the
      66             :  * `shape_function_of_time_name` argument in the constructor that must always be
      67             :  * specified) are ***not*** the complex spherical-harmonic coefficients
      68             :  * \f$\lambda_{lm}(t)\f$, but instead are the real-valued SPHEREPACK

      70             :  * is the same for both methods of specifying FunctionOfTime%s above. The
      71             :  * relationship between these two sets of coefficients is
      72             :  * \f{align}
      73             :  * a_{l0} & = \sqrt{\frac{2}{\pi}}\lambda_{l0}&\qquad l\geq 0,\\
      74             :  * a_{lm} & = (-1)^m\sqrt{\frac{2}{\pi}} \mathrm{Re}(\lambda_{lm})
      75             :  * &\qquad l\geq 1, m\geq 1, \\
      76             :  * b_{lm} & = (-1)^m\sqrt{\frac{2}{\pi}} \mathrm{Im}(\lambda_{lm})
      77             :  * &\qquad l\geq 1, m\geq 1.
      78             :  * \f}
      79             :  * The "shape" FunctionOfTime stores coefficients only for non-negative \f$m\f$;
      80             :  * this is because the function we are expanding is real, so the
      81             :  * coefficients for \f$m<0\f$ can be obtained from \f$m>0\f$ coefficients by
      82             :  * complex conjugation.
      83             :  * If the `size_function_of_time_name` argument is given to the constructor,
      84             :  * then it is asserted that the \f$l=0\f$ coefficient of the "shape" function of
      85             :  * time is exactly 0. The \f$l=0\f$ coefficient is then controlled by the "size"
      86             :  * FunctionOfTime. Unlike the "shape" FunctionOfTime, the quantity in the
      87             :  * "size" FunctionOfTime ***is*** the "complex" spherical harmonic coefficient
      88             :  * \f$\lambda_{00}(t)\f$, and not the SPHEREPACK coefficient \f$a_{00}(t)\f$
      89             :  * ("complex" is in quotes because all \f$m=0\f$ coefficients are always real.)
      90             :  * Here and below we write the equations in terms of \f$\lambda_{lm}(t)\f$
      91             :  * instead of \f$a_{lm}(t)\f$ and \f$b_{lm}(t)\f$, regardless of which
      92             :  * FunctionOfTime representation we are using, because the resulting expressions
      93             :  * are much shorter.
      94             :  *

      97             :  * zero at the boundary of the domain.
      98             :  *
      99             :  * ### Mapped coordinates
     100             :  *
     101             :  * The shape map maps the unmapped
     102             :  * coordinates \f$\xi^i\f$ to coordinates \f$x^i\f$:
     103             :  *

     105             :  * x^i = \xi^i - (\xi^i - x_c^i) f(r, \theta, \phi) \sum_{lm}
     106             :  * \lambda_{lm}(t)Y_{lm}(\theta, \phi).
     107             :  * \f}
     108             :  *
     109             :  * ### Inverse map
     110             :  *
     111             :  * The inverse map is given by:
     112             :  * \f{equation}{
     113             :  * \xi^i = x_c^i + (x^i-x_c^i)*(r/\tilde{r}),
     114             :  * \f}
     115             :  * where \f$\tilde{r}\f$ is the radius of \f$\xi\f$, calculated by the
     116             :  * transition map. For more details, see
     117             :  * ShapeMapTransitionFunction::original_radius_over_radius .
     118             :  *
     119             :  * ### Frame velocity
     120             :  *
     121             :  * The frame velocity \f$v^i\ = dx^i / dt\f$ is calculated trivially:
     122             :  * \f{equation}{
     123             :  * v^i = - (\xi^i - x_c^i) f(r, \theta, \phi) \sum_{lm}
     124             :  * \dot{\lambda}_{lm}(t)Y_{lm}(\theta, \phi).
     125             :  * \f}
     126             :  *
     127             :  * ### Jacobian
     128             :  *
     129             :  * The Jacobian is given by:
     130             :  * \f{equation}{
     131             :  * \frac{\partial}{\partial \xi^j} x^i = \delta_j^i \left( 1 - f(r, \theta,
     132             :  * \phi) \sum_{lm} \lambda_{lm}(t)Y_{lm}(\theta, \phi)\right) + (\xi^i - x_c^i)
     133             :  * \left(\frac{\partial}{\partial \xi^j} f(r, \theta, \phi) \sum_{lm}
     134             :  * \lambda_{lm}(t)Y_{lm}(\theta, \phi) + f(r, \theta, \phi) \sum_{lm}
     135             :  * \lambda_{lm}(t) \frac{\partial}{\partial \xi^j} Y_{lm}(\theta, \phi) \right).
     136             :  * \f}
     137             :  *
     138             :  * ### Inverse Jacobian

     149             :  */
