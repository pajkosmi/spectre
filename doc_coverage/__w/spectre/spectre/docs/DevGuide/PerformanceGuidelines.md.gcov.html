       9             : Below are some general guidelines for achieving decent performance.
      10             : 
      11             : - One good measurement is worth more than a million expert opinions. We have a
      12             :   `Benchmark` executable that uses Google Benchmark so one can compare different
      13             :   implementations and see how they perform. This executable is only available in
      14             :   release builds.
      15             : - Reduce memory allocations. On all modern hardware (many core CPUs, GPUs, and
      16             :   FPGAs), memory is almost always the bottleneck. Memory allocations are
      17             :   especially expensive since this is a quasi-serial process: the OS has to
      18             :   manage memory allocations for _all_ running threads and processes. SpECTRE has
      19             :   various classes to optimize this. For example, there are `Variables`,
      20             :   `TempBuffer`, and `DynamicBuffer` that allow making large contiguous memory
      21             :   allocations that are then used for individual tensor components.
