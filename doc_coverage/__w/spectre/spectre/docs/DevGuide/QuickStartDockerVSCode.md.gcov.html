       5             : # Code development quick-start with Docker and Visual Studio Code {#dev_guide_quick_start_docker_vscode}
       6             : 
       7             : \tableofcontents
       8             : 
       9             : This page describes how to get started developing SpECTRE on Mac, Linux, or
      10             : Windows using [Docker](https://docker.com) and the [Visual Studio
      11             : Code](https://code.visualstudio.com) editor. This is a particularly quick way to
      12             : get up and running with SpECTRE code development, though of course not the only
      13             : way. If you prefer setting up your development environment differently, we
      14             : suggest you read the \ref installation "Installation" page. If you would like to
      15             : jump right into a working development environment, read on!
      16             : 
      17             : ## Fork the SpECTRE repository on GitHub
      18             : 
      19             : The SpECTRE code lives on GitHub in the
      20             : [sxs-collaboration/spectre](https://github.com/sxs-collaboration/spectre)
      21             : repository. Developers work on their own copy (or
      22             : ["fork"](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/about-forks))
      23             : of the repository and contribute back to
      24             : [sxs-collaboration/spectre](https://github.com/sxs-collaboration/spectre)
      25             : through [pull requests](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/about-pull-requests).
      26             : Fork the repository to your own account:
      27             : 
      28             : - [Fork sxs-collaboration/spectre on GitHub](https://github.com/sxs-collaboration/spectre/fork)
      29             : 
      30             : ## Clone the SpECTRE repository to your computer
      31             : 
      32             : To work on SpECTRE code you will need a local copy (or "clone") of the
      33             : repository on your computer. We use SSH to communicate with GitHub, so you need
      34             : to set up your SSH keys first. Follow GitHub's instructions to generate an SSH
      35             : key and add it to your GitHub account:
      36             : 
      37             : - [Generating an SSH key and adding to to GitHub](https://docs.github.com/en/github/authenticating-to-github/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent)
      38             : 
      39             : Now you can download the repository via SSH. Navigate to **your fork** of the
      40             : SpECTRE repository (i.e. the repository at the URL
      41             : https://github.com/YOURNAME/spectre). Follow GitHub's instructions to clone the
      42             : repository to your computer, selecting the SSH option:
      43             : 
      44             : - [Cloning a repository](https://docs.github.com/en/github/creating-cloning-and-archiving-repositories/cloning-a-repository)
      45             : 
      46             : \note If you clone the SpECTRE repository using HTTPS instead of SSH, you won't
      47             : be able to push any changes you make to your fork. You'll still be able to make
      48             : local changes on your computer, but you won't be able to share your changes with
      49             : anybody.
      50             : 
      51             : ## Enable the development environment in the repository
      52             : 
      53             : The development environment is included in the repository, but not enabled by
      54             : default. To enable it, copy or symlink the directory
      55             : `support/DevEnvironments/.devcontainer` to the repository root. This is easiest
      56             : done with the command line. Navigate into the repository that you just cloned to
      57             : your computer:
      58             : 
      59             : ```
      60             : cd spectre
      61             : ```
      62             : 
      63             : Now symlink the development environment:
      64             : 
      65             : ```
      66             : ln -s support/DevEnvironments/.devcontainer .devcontainer
      67             : ```
      68             : 
      69             : ## Install Docker
      70             : 
      71             : On your computer you will need Docker to run the containerized development
      72             : environment. Install and start Docker:
      73             : 
      74             : - [Get Docker](https://docs.docker.com/get-docker/)
      75             : 
      76             : If you're new to Docker, you can read through Docker's [Getting
      77             : started](https://docs.docker.com/get-started/) documentation to learn about
      78             : their basic concepts. We will use Docker to download and jump into a prebuilt
      79             : development environment that has everything installed to compile and run
      80             : SpECTRE.
      81             : 
      82             : ## Install Visual Studio Code
      83             : 
      84             : We will use the Visual Studio Code editor to jump into the containerized
      85             : development environment, edit code, compile executables and run them. Download
      86             : and install Visual Studio Code:
      87             : 
      88             : - [Get Visual Studio Code](https://code.visualstudio.com)
      89             : 
      90             : Microsoft maintains [extensive
      91             : documentation](https://code.visualstudio.com/docs) for Visual Studio Code that
      92             : you can read to get started using the editor. We recommend you take a look
      93             : through the [Tips and
      94             : Tricks](https://code.visualstudio.com/docs/getstarted/tips-and-tricks) to get an
      95             : idea of the editor's features.
      96             : 
      97             : ## Install the "Remote - Containers" extension
      98             : 
      99             : Visual Studio Code's "Remote - Containers" extension lets you run the editor in
     100             : a Docker container. Install the extension:
     101             : 
     102             : - [Install the "Remote - Containers"
     103             :   extension](vscode:extension/ms-vscode-remote.remote-containers)
     104             : 
     105             : ## Open the SpECTRE repository in Visual Studio Code
     106             : 
     107             : Now open the SpECTRE repository that you have cloned to your computer in Visual
     108             : Studio Code. Depending on your operating system you can select `File > Open`,
     109             : type `Cmd+O` (macOS) or `Ctrl+O` (Linux or Windows), drag the repository folder
     110             : onto the Visual Studio Code icon or any other means to open the folder in Visual
     111             : Studio Code.
     112             : 
     113             : Now is also time to learn how to use the single most important tool in Visual
     114             : Studio Code, the command palette. Try it now: Hit `Cmd+P` (macOS) or `Ctrl+P`
     115             : (Linux or Windows) and start typing the name of any file, for example
     116             : `QuickStartDockerVSCode.md`. Hit `Enter` to open the file. This is how you can
     117             : quickly open any file in the repository. Note that the search is fuzzy, so you
     118             : can type any few letters in the path to the file. In addition to opening files,
     119             : you can hit `Cmd+Shift+P` (macOS) or `Ctrl+Shift+P` (Linux or Windows) and type
     120             : the name of any command that Visual Studio Code supports (or parts of it), for
     121             : example `Preferences: Open User Settings`.
     122             : 
     123             : ## Reopen the SpECTRE repository in the development container
     124             : 
     125             : Open the command palette by hitting `Cmd+Shift+P` (macOS) or `Ctrl+Shift+P`
     126             : (Linux or Windows) and run the command `Remote-Containers: Reopen in container`
     127             : by starting to type a few letters of this command and then hitting `Enter`.
     128             : 
     129             : Visual Studio Code will download and run the container and drop you into a fully
     130             : configured environment where you can proceed to compile and run SpECTRE. This
     131             : may take a few minutes so just let VSCode run and do it's thing. There may be a
     132             : `show logs` button somewhere on your screen if you're interested in the details
     133             : of the download.
     134             : 
     135             : If you are interested to learn more about how this feature works you can
     136             : read through the [VS Code documentation on remote containers](https://code.visualstudio.com/docs/remote/containers)
     137             : and inspect the `.devcontainer/devcontainer.json` file that's included in the
     138             : SpECTRE repository.
     139             : 
     140             : \note If you look into the `.devcontainer/devcontainer.json` file, you'll see we
     141             : are using the `sxscollaboration/spectre:dev` container. We also offer other
     142             : containers. The `sxscollaboration/spectre:ci` container has more compilers and
     143             : the `sxscollaboration/spectre:demo` container has a couple pre-built
     144             : executables. If you'd like to build in a different container, just switch the
     145             : image name in the json file and reopen in the new container.
     146             : 
     147             : ## Configure, compile and run SpECTRE
     148             : 
     149             : With Visual Studio Code running in the development container you can now
     150             : configure, compile and run SpECTRE with no additional setup. Hit `Cmd+Shift+P`
     151             : (macOS) or `Ctrl+Shift+P` (Linux or Windows) to open the command palette and run
     152             : the command `CMake: Select a Kit`. You should see a `Default` kit available.
     153             : Then, open the command palette again, run `CMake: Select Variant`, and select
     154             : either `Release` or `Debug`. For this tutorial we recommend using `Release`. If
     155             : you want to actually develop SpECTRE, the `Debug` option would be better. This
     156             : specific kit uses `clang-10` to build SpECTRE along with the Python bindings.
     157             : Finally, open the command palette once more and run `CMake: Configure` which
     158             : will now actually run CMake.
     159             : 
     160             : It will set up a build directory. You can open
     161             : Visual Studio Code's [integrated
     162             : terminal](https://code.visualstudio.com/docs/editor/integrated-terminal) with
     163             : the keyboard shortcut ``Ctrl+` `` and navigate to the newly created build
     164             : directory to inspect it:
     165             : 
     166             : ```
     167             : cd build-Default-Debug
     168             : ```
     169             : 
     170             : For compiling and running SpECTRE you can either use Visual Studio Code's CMake
     171             : integration by looking up further commands in the command palette, or use the
     172             : terminal. We will be using the terminal from now on. Try compiling an
     173             : executable, for example:
     174             : 
     175             : ```
     176             : make -j4 ExportCoordinates3D
     177             : ```
     178             : 
     179             : Once the executable has compiled successfully you can try running it:
     180             : 
     181             : ```
     182             : ./bin/ExportCoordinates3D \
     183             :   --input-file $SPECTRE_HOME/tests/InputFiles/ExportCoordinates/Input3D.yaml
     184             : ```
     185             : 
     186             : This executable produced a volume data file in the `build-Default-Debug`
     187             : directory that we can visualize in ParaView. Generate an XMF file from the
     188             : volume data file that ParaView understands:
     189             : 
     190             : ```
     191             : ./bin/spectre generate-xdmf \
     192             :   --file-prefix ExportCoordinates3DVolume --subfile-name element_data \
     193             :   --output ExportCoordinates3DVolume
     194             : ```
     195             : 
     196             : Since the build directory is shared with the host file system you can now open
     197             : ParaView on your computer and load the generated XMF file as described in the
     198             : \ref tutorial_visualization "visualization tutorial".
     199             : 
     200             : \note If you wanted to run this executable again, you'd have to first remove the
     201             : existing `.h5` files, otherwise an error will occur.
     202             :

     224             : ## Interactively debug a SpECTRE test with VS Code
     225             : 
     226             : To track down an issue with your code it can be very useful to interactively
     227             : debug it. First, make sure you have written a test case where the issue occurs.
     228             : If you have not written a test yet, this is a great time to do it. Refer to the
     229             : \ref writing_unit_tests "Writing Unit Tests" dev guide to learn how to write
     230             : tests.
     231             : 
     232             : Now configure the `RunSingleTest` executable to run your particular test so you
     233             : don't have to repeatedly compile the extensive `RunTests` executable. Also refer
     234             : to the \ref writing_unit_tests "Writing Unit Tests" dev guide for this.
     235             : 
     236             : To launch the interactive debugger, hit `Cmd+Shift+P` (macOS) or `Ctrl+Shift+P`
     237             : (Linux or Windows) to open the command palette, run the command `CMake: Set
     238             : Debug Target` and select `RunSingleTest`. Then run the command `CMake: Debug`.
     239             : The test executable will compile, run and stop on any breakpoints. Follow the
     240             : Visual Studio Code documentation to learn how to set breakpoints in your code
     241             : and inspect the state of your variables:
     242             : 
     243             : - [Debug actions in VS Code](https://code.visualstudio.com/docs/editor/debugging#_debug-actions)
     244             :

     256             : ## Tips and tricks
     257             : 
     258             : - The [**GitLens extension**](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens)
     259             :   is very useful to browse your repository. Select the "Source Control" icon and
     260             :   explore the various panels.
     261             : 
     262             : - When you build the **documentation** (e.g. with `make doc`), you can open it
     263             :   in a web server within VS Code:
     264             : 
     265             :   ```
     266             :   python3 -m http.server -d docs/html
     267             :   ```
     268             : 
     269             :   The web server launches on port 8000 by default, which is being forwarded
     270             :   outside the container, so you can just open http://127.0.0.1:8000 in your
     271             :   browser to view the documentation.
     272             : 
     273             : - Instead of working in the Docker container, you can use the [Remote - SSH](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-ssh)
     274             :   extension to **work on a remote machine** such as your local supercomputing
     275             :   cluster. Ask the cluster administrators or users for suggestions concerning
     276             :   installing and running SpECTRE on the particular supercomputer.
     277             : 
     278             : - You can work with **Jupyter notebooks in Visual Studio Code**. First, install
     279             :   Jupyter and all other Python packages you want to work with in the container:
     280             : 
     281             :   ```
     282             :   pip3 install jupyter matplotlib
     283             :   ```
     284             : 
     285             :   Any `.ipynb` notebook files you create or open will be displayed in VS Code's
     286             :   notebook editor. This is very useful for quickly plotting data from SpECTRE
     287             :   runs or using SpECTRE's Python bindings. Refer to the [VS Code documentation
     288             :   on Jupyter support](https://code.visualstudio.com/docs/python/jupyter-support)
     289             :   for more information.
     290             : 
     291             : - Docker can quickly use up a lot of disk space. From time to time you
     292             :   can "prune" unneeded images and containers to reclaim disk space:
     293             : 
     294             :   - [Prune unused Docker objects](https://docs.docker.com/config/pruning/)
     295             : 
     296             :   When pruning containers, make sure no data is deleted that you care about!
