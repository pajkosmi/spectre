      33             :  * The volume data inside the subfile can be of any dimensionality greater than

      42             :  *
      43             :  * \par Grid names
      44             :  * The data stored in the subfile are the tensor components passed to the

      46             :  * the `GRID_NAME` should be the output of the stream operator of the spatial ID
      47             :  * of the parallel component element sending the data to be observed. For
      48             :  * example, in the case of a dG evolution where the spatial IDs are
      49             :  * `ElementId`s, the grid names would be of the form `[B0,(L2I3,L2I3,L2I3)]`.
      50             :  *
      51             :  * \par Data layout in the H5 file
      52             :  * The data are written contiguously inside of the H5 subfile, in that each
      53             :  * tensor component has a single dataset which holds all of the data from all
      54             :  * elements, e.g. a tensor component `T_xx` which is found on all grids appears
      55             :  * in the path `H5_FILE_NAME/SUBFILE_NAME.vol/OBSERVATION_ID/GRID_NAME/T_xx` and

      57             :  * coordinates must be written as tensors in order to visualize the data in
      58             :  * ParaView, Visit, etc.  In order to reconstruct which data came from which
      59             :  * grid, the `get_grid_names()`, and `get_extents()` methods list the grids

      62             :  * `{2, 2, 2}`, it was responsible for contributing the first 2*2*2 = 8 grid
      63             :  * points worth of data in each tensor dataset. Use the
      64             :  * `h5::offset_and_length_for_grid` function to compute the offset into the
      65             :  * contiguous dataset that corresponds to a particular grid.
      66             :  *
      67             :  * \par Domain and FunctionsOfTime
      68             :  * A serialized representation of the domain and the functions of time can be
      69             :  * written into the subfile alongside the tensor data. Reconstructing the domain

      71             :  * tensor data to another set of points. The domain is currently stored as a
      72             :  * `std::vector<char>`, and it is up to the calling code to serialize and
